数据加载

数据加载主要使用两个类：

  • Dataset：负责将数据文件从操作系统的文件系统（外存）中读取出来；
  • Dataloader：负责将数据文件组织成一定的形式，例如batch，用来训练；
    这两个类共同实现了将存储在文件系统中的数据，提供给模型进行训练的功能，属于前后协作；

此外一个数据一般有：

  • 数据本身；
  • label；
    对于前者，不同形式的数据还需要不同的手段进行处理，例如：
  1. 图像数据，可以使用PLI的image包或者一些cv的包
  2. 文本数据，需要一些文本处理的包
    对于label，通常采用整型 or 字符串等，处理起来比较简单；

数据加载的目标

目标可以概括为：

  • 针对自己的数据，实现Dataset的相应子类；
  • 针对具体情况，使用Dataloader；
    （因为Dataset类中规定了继承自己的子类必须重写的函数是什么，当你完成自己的子类的时候，再和Dataloader结合使用，就意味着你有了可以用于加载自己的数据的“工具”，这就是数据加载在实现层面的目标）；

数据集的文件组织形式

数据集文件的组织形式意味着你在编程时的处理范式，一种类型的组织形式往往有着较为固定的编程范式。

按照label所在的位置，主要有三种形式：

  1. label作为数据文件存储的文件夹名
  • trian/label_name/…(存储的数据文件)
  • val/label_name/…(存储的数据文件)
  1. label作为数据文件的文件名
  • train/label_name(对应存储的数据文件)
  • val/label_name(对应存储的数据文件)
  1. label单独存储为一个txt文件
  • train/001(编号对应的数据文件)
  • train_label/001(编号对应的label)

下载使用的数据集

下面的代码按照第一种形式组织；

"""
读取数据集的模块：DataReader.read_data_insect
"""
from torch.utils.data import Dataset
from PIL import Image
import os

class data_reader_insect(Dataset):
    def __init__(self, root_dir, label_dir): # 初始化的时候将数据的地址组织成一个列表
        self.root_dir = root_dir
        self.dir_list = os.listdir(os.path.join(root_dir,label_dir))
        self.label_dir = label_dir
    def __getitem__(self,idx): # 使用idx访问列表，得到图片地址，读取到图片
        img_dir = self.dir_list[idx]
        img_dir = os.path.join(self.root_dir,self.label_dir,img_dir) # os.listdir只返回文件名，需要补全路径！！！
        img = Image.open(img_dir)
        label = self.label_dir
        return img,label # 最后返回数据和标签
    def __len__(self):
        return len(self.dir_list)

注意事项

  1. py项目中如果要把一个文件夹当中模块，那么文件夹中必须要有__init__.py，即使里面什么都不写，当作模块才方便其他py文件用import的方式进行代码复用；
  2. os.listdir(文件夹路径)，是用列表的形式返回文件夹下的文件名，在构造数据文件路径的时候要用os.path.join()对路径进行拼接；
  3. py中函数return多个对象时，是用元组的形式返回的
  4. 能够使用train_data[idx]的形式访问数据的原因是对于Dataset类型会直接解析为调用Dataset.__getitem__(self,index)函数

训练过程可视化：TensorBoard

TensorBoard作为一个训练的可视化工具，可以帮我们把整个训练过程中产生的一些数据收集起来，形成图像，帮助我们快速直观地了解整个训练过程；

SummaryWriter

SummaryWriter是Pytorch提供的工具类，从而让torch可以结合TensorBoard使用：

  1. 该类可以向日志文件夹中写入event文件，该文件可以被TensorBoard解析，从而形成可视化的结果；
  2. 日志可以通过SummaryWriter的初始化参数指定，如果不指定的话默认创建一个名为runs的文件夹使用

SummaryWriter类常用的方法

  1. add_scalar()
  • 原型：def add_scalar(self, tag, scalar_value, global_step=None, walltime=None)
  • sclar_value，(x,y)中的y，即步-值中的值
  • global_step，(x,y)中的x，即步-值中的步
  • tag(String)，对应图表的标题，通常一个tag会对应一个event文件，即修改tag重新绘制会创建一个新的event文件
  • 如果修改了绘图方式，但是不修改tag，直接运行，两次截然不同的运行结果被写入同一个event文件，tensorboard绘制会出现问题；
  • 官方建议是在有不修改tag，但是要更换绘制方式的需求时，先删除原始的event文件
  1. add_image()
  • 原型：def add_image(self, tag, img_tensor, global_step=None, walltime=None, dataformats='CHW')
  • img_tensor，它对应的就是图像数据，主要有torch.tensor和numpy.array两种类型
  • 图像转换为张量之后通常有高H，宽W，通道C，三个维度，并且通道数C一般是3（rgb）；
  • 该方法默认图像张量是$C\times H\times W$，从参数默认值也可以看出来
  • 如果张量不对应需要手动指定dataformats，例如用PIL的Image打开的图像，由numpy.array()方法转出的图像默认是$H\times W\times C$
  1. close()
  • 在完成写入文件之后使用，关闭IO流

基本使用方法

  1. 创建SummaryWriter对象
  2. 使用相关方法写入日志文件夹中的事件文件
  3. 在命令行使用tensorboard --logdir=日志文件夹名
  • 默认会在本地的6006端口打开一个网页，显示TensorBoard根据事件文件的绘制内容（会绘制所有的事件文件对应的图）；
  • 如果在一个服务器上，多个人都使用tensorboard的话默认端口可能有冲突，可以使用tensorboard --logdir=日志文件夹名 --port==端口号，指定端口；

Transforms

pytorch中用于处理图像的工具（一个py模块），里面有很多好用的工具类

导入方式是：

from torchvision import transforms

ToTensor

这是最常用的类，主要作用是将PIL Image或numpy的对象转换为tensor类型；
（PIL的Image类图像得到Image对象，cv包处理图像得到numpy对象，这是两种常见的图像处理方式）

tensor数据类型是pytorch框架下的一个重要数据类型，几乎所有使用pytorch实现的，涉及模型、网络的相关代码都有这个类型；

关注两个问题：

  1. 为什么要使用tensor数据类型
  2. transform如何使用（以ToTensor为例）

为什么要使用tensor

tensor本身就是pytorch针对神经网络的训练实现的一个数据类型，该类中不仅可以存储与模型、网络相关的数据，还集成了许多训练时所需的属性、方法；

Transform的使用方法

  1. 直接导入其中的工具类，实例化为对象进行使用；
  2. 将其中的某个工具类导入进来，然后创建一个自己的类对其进行继承；即基于一个模板，实现自己针对特定任务所需的趁手工具；

常见的Transforms

对于相关的工具类关注三个点：

  1. 输入
  2. 输出
  3. 作用

为什么要关注输入和输出？

  • 因为图片有不同的格式
  • 例如PIL是用python自带的Image.open()打开的类型
  • tensor则是用ToTensor()
  • 还有narrays矩阵，使用OpenCV库中的cv.imread()

Compose

Compose主要关注作用，具体的输入输出视其中的具体transforms对象而定

（在我们说transforms对象的时候一般指transforms.py中的类的对象）

导入：

from torchvision.transforms import Compose

或者可以：

from torchvision import transforms
transforms.Compose(...)

注意Compose是transforms.py中的一个class，所以Compose(…)的使用方法实际上是使用了如下所示的call方法。该方法允许直接将类的对象，当作一种函数使用。

class Compose(object):
    #...
    def __call__(self, img):
    #...

Compose的作用是将不同的transforms（图像处理操作）结合在一起，例如

mytrans = transforms.Compose([
    transforms.CenterCrop(10),
    transforms.ToTensor(),
])

定义好的 transforms.Compose 流水线本质上是一个可调用的对象（callable），你可以直接将它应用到图像数据上。常见的用法如下：

from PIL import Image
from torch.utils.torchvision import transforms

# 定义流水线
transform = transforms.Compose([
    transforms.CenterCrop(10),
    transforms.ToTensor(),
])

# 加载图像
img = Image.open('example.jpg')  # PIL Image
# 应用变换
img_tensor = transform(img)      # 输出是形状为 (C, H, W) 的张量，像素值在 [0,1]

输入与输出

  1. 输入是PIL Image或者是numpy.ndarray
  2. 输出是转换后的tensor类型

作用

  • 将其他的图片类型转换为tensor的类型
from PIL import Image

Normalize

输入与输出

  1. 输入是tensor的图片以及均值mean（对于n个channel对应的是(M1,M2,…,Mn)）和标准差std（对于n个channel对应的是(S1,S2,…,Sn)），最终
  2. 输出的是处理后的原tensor图片，处理的方式是input[channel] = (input[channel] - mean[channel])/std[channel]

作用

  • 将输入的图片按照指定的均值和标准差进行归一化处理，即对于图片每个通道的像素值进行对应的变换；

例如，在下面的代码中我们会发现normal_img[0][0][0] = (tensor_img[0][0][0] - 0.5)/0.5

from PIL import Image
from torchvision import transforms
from torch.utils.tensorboard import SummaryWriter

img = Image.open(...)
totensor = transforms.ToTensor()
tensor_img = totensor(img)
writer = SummaryWriter("logs")

print(tensor_img[0][0][0])
writer.add_image("before normalize",tensor_img,0)


normalizer = transforms.Normalize([0.5,0.5,0.5],[0.5,0.5,0.5])
normal_img = normalizer(tensor_img)

print(normal_img[0][0][0])
wirter.add_image("after normalize",tensor_img,1)

writer.close()

注意：

  1. 从代码中可以看出，具体的means和std是在Normalize对象创建的时候指定的；
  2. 而使用的时候是通过call方法，即不让对象调用具体的函数；

Resize

输入与输出

  1. 输入
  • 一个尺寸，例如一个尺寸序列(h,w)，那么图片就会被按照这个高:宽的尺寸进行调整；如果只给定一个数字number，那么图片的最小边将会匹配这个数字，类似于等比缩放；
  • 输入是在对象创建的时候进行的；
  1. 输出
  • 尺寸改变后的图像，一般使用也是通过call方法；
  • 在使用call方法的时候除了tensor类型的图像，还可以直接接收PIL Image的图像

作用

  • 改变图像的尺寸

使用序列作为Resize的初始化参数例子如下

from PIL import Image
from torchvision import transform
from torch.utils.tensorboard import SummaryWriter

img = Image.open(...)
resizer = transform.Resize((512,512)) #注意这种序列的写法
writer = SummaryWriter("logs")
totensor = transforms.ToTensor()

tensor_img = totensor(img)
writer.add_image("before resize",tensor_img,0)

print(img.size)
resized_img = resizer(img)
print(resized_img.size)

tensor_img = totensor(resized_img)
writer.add_image("after resize",tensor_img,1)
writer.close()

由于Resize可以直接处理PIL Image的图片，所以将它与ToTensor结合到Compose中是一个很好的Compose的例子，下面以单个参数，等比缩放为例

from PIL import Image
from torchvision import transforms
from tensorboard import SummaryWriter

img = Image.open(...)
resizer = transforms.Resize(512)
totensor = transforms.ToTensor()
mytrans = transforms.Compose([resizer,totensor])

final_img = mytrans(img)
writer.add_img("Compose",final_img)
writer.close()

RandomCrop

输入与输出

  1. 输入（对象创建时）
  • 一般在创建对象时输入一个size，类似于Resize（可以是数字、序列）
  1. 输出
  • 接收一个图片，输出一张从原图像上按照size大小裁剪出来的图片

作用

  • 按照指定大小与随机的一个位置，从原始的图像上裁剪下来一部分“小”的图片；
from PIL import Image
from torchvision import transforms
from torch.utils.tensorboard import SummaryWriter

img = Image.open(...)
trans_random = transforms.RandomCrop(512)
totensor = transforms.ToTensor()
writer = SummaryWriter("logs")

tensor_img = totensor(img)
for i in range(10):
    rand_img = trans_random(tensor_img)
    writer.add_image("RandomCrop",rand_img,i)

writer.close()

将数据集与transforms结合（datasets的使用）

上面常用的transform讲解了各种处理图像的工具类大概是怎么使用的，但是示范的时候都是面向一张图片的，在实际当中我们肯定是面对一整个数据集来进行处理的。这就是这部分要研究的内容。

主要的结合思路是，定义自己的数据集，然后使用Compose（自己指定的一系列变换）作为数据集的transform

例如

import torchvision

mytrans = torchvision.transforms.Compose([
    torchvision.transforms.ToTensor()
])
train_sets = (root = "./test", train = True, transform = mytrans, Download = True)
test_sets = (root = "./test", train = False, transfrom = mytrans, Download = True)

print(train_sets[3]) # 会发现打印的是一系列矩阵中的数字（tensor），和最后的一个label数字

可以结合tensorboard一起使用

import torchvision
from torch.utils.tensorboard import SummaryWriter

mytrans = torchvision.transforms.Compose([
    torchvision.transforms.ToTensor()
])
train_sets = (root = "./test", train = True, transform = mytrans, Download = True)
test_sets = (root = "./test", train = False, transfrom = mytrans, Download = True)

print(train_sets[3]) # 会发现打印的是一系列矩阵中的数字（tensor），和最后的一个label数字

writer = SummaryWriter("logs_cifar")
for i in range(10):
    img, label = train_sets[i]
    writer("cifar_trains", img, i)

writer.close()

注意：具体的数据集要设置的参数是不同的，但是整体来说是大同小异的；

主要关注如下几个目录：

  1. root，存放各种数据集的根目录，指定后会到对应的目录寻找后缀.gz的压缩文件 or 解压好的文件（如果没有解压好会自动校验、解压）；
  2. train，如果设置为True代表是训练集，如果设置为False代表是测试集，压缩文件中有专用于训练和测试的数据集，这个参数影响具体使用的哪个数据集；
  3. Download，设置为True，如果root下没有对应的数据集会自动下载压缩包，并完成解压；
  4. transform，一般是利用Compose，定义好对数据集图片中的预处理；
  5. 对于torchvision.datasets的对象，可以当作数组访问，每个元素一般是一个二元组，对应一张图片和一个label
  6. torchvision.datasets的对象的classes属性中有每个label对应的具体标签是什么

dataloader

datasets本质上是告诉程序我们数据集的位置，以及提供一些方式访问到对应数据的具体信息（图片、label...）。但是我们需要一个更加贴近为训练服务（例如将数据加载到神经网络）的工具，这就是dataloader

具体的导入方式如下

from torch.utils.data import DataLoader

DataLodaer类的一些重要的参数

  • dataset，用于指定从哪里加载数据；
  • batch_size，指定每次加载多少数据，例如两张图片；
  • shuffle，如果为True，每次加载前数据集的顺序会被打乱；
  • num_workers，表示使用多少个进程加载数据，默认为0代表使用主进程加载数据；
  • drop_last，为True的话，如果最后不够batch_size则会舍弃最后的数据；

dataloader的功能

  • 从datasets中取数据，具体怎么取由相关的参数确定
  • 在最常用的参数下（下方的例子），dataloader会将datasets中的数据按照images，labels的双元组进行打包，其中images是图片的列表、labels是label的列表
  • 我们通过类似some_object_of_datasets[0]的方式访问datasets返回的就是image和label的双元组
import torchvision
from torch.utils.data import DataLoader

test_sets = torchvision.datasets.CIFAR10(root="./DataSets/CIFAR10", train = False, transform = torchvision.transforms.ToTensor(),download=False)
test_loader = DataLoader(dataset=test_sets, batch_size=4, shuffle=True, drop_last=False, num_workers=0)

for data in test_loader:
    imgs, labels = data
    print(imgs.shape)
    print(labels)

上例输出如下

...
torch.Size([4, 3, 32, 32])
tensor([4, 6, 7, 9])
torch.Size([4, 3, 32, 32])
tensor([3, 3, 2, 3])

对应的就是4张3通道32*32的图片、对应4个label分别是4 6 7 9

注意，上例的数据是随机取的，因为另一个最重要的参数，“采样器” sampler没有设置；

sampler默认是torch.utils.data.sampler.RandomSampler，即随机采样的采样器

最后引入一个epoch的概念，通常以epoch为单位对网络进行一次训练，一次epoch可能会用到多个batch，如下面的代码所示

import torchvision
from torch.utils.data import DataLoader
from torch.utils.tensorboard import SummaryWriter

test_sets = torchvision.datasets.CIFAR10(root="./DataSets/CIFAR10", train = False, transform = torchvision.transforms.ToTensor(),download=False)
test_loader = DataLoader(dataset=test_sets, batch_size=4, shuffle=True, drop_last=False, num_workers=0)
writer = SummaryWriter("./logs/dataloader_logs")

for epoch in range(10):
    step = 0
    for data in test_loader:
        imgs, labels = data
        print(imgs.shape)
        print(labels)
        writer.add_images("test_images", imgs, global_step=step)
        step += 1

在实际训练的时候，上面对于dataloader的循环下就是对于网络的操作，会使用imgs、labels作为输入
