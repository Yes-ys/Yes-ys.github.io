数据加载

数据加载主要使用两个类：

  • Dataset：负责将数据文件从操作系统的文件系统（外存）中读取出来；
  • Dataloader：负责将数据文件组织成一定的形式，例如batch，用来训练；
    这两个类共同实现了将存储在文件系统中的数据，提供给模型进行训练的功能，属于前后协作；

此外一个数据一般有：

  • 数据本身；
  • label；
    对于前者，不同形式的数据还需要不同的手段进行处理，例如：
  1. 图像数据，可以使用PLI的image包或者一些cv的包
  2. 文本数据，需要一些文本处理的包
    对于label，通常采用整型 or 字符串等，处理起来比较简单；

数据加载的目标

目标可以概括为：

  • 针对自己的数据，实现Dataset的相应子类；
  • 针对具体情况，使用Dataloader；
    （因为Dataset类中规定了继承自己的子类必须重写的函数是什么，当你完成自己的子类的时候，再和Dataloader结合使用，就意味着你有了可以用于加载自己的数据的“工具”，这就是数据加载在实现层面的目标）；

数据集的文件组织形式

数据集文件的组织形式意味着你在编程时的处理范式，一种类型的组织形式往往有着较为固定的编程范式。

按照label所在的位置，主要有三种形式：

  1. label作为数据文件存储的文件夹名
  • trian/label_name/…(存储的数据文件)
  • val/label_name/…(存储的数据文件)
  1. label作为数据文件的文件名
  • train/label_name(对应存储的数据文件)
  • val/label_name(对应存储的数据文件)
  1. label单独存储为一个txt文件
  • train/001(编号对应的数据文件)
  • train_label/001(编号对应的label)

下载使用的数据集

下面的代码按照第一种形式组织；

"""
读取数据集的模块：DataReader.read_data_insect
"""
from torch.utils.data import Dataset
from PIL import Image
import os

class data_reader_insect(Dataset):
    def __init__(self, root_dir, label_dir): # 初始化的时候将数据的地址组织成一个列表
        self.root_dir = root_dir
        self.dir_list = os.listdir(os.path.join(root_dir,label_dir))
        self.label_dir = label_dir
    def __getitem__(self,idx): # 使用idx访问列表，得到图片地址，读取到图片
        img_dir = self.dir_list[idx]
        img_dir = os.path.join(self.root_dir,self.label_dir,img_dir) # os.listdir只返回文件名，需要补全路径！！！
        img = Image.open(img_dir)
        label = self.label_dir
        return img,label # 最后返回数据和标签
    def __len__(self):
        return len(self.dir_list)

注意事项

  1. py项目中如果要把一个文件夹当中模块，那么文件夹中必须要有__init__.py，即使里面什么都不写，当作模块才方便其他py文件用import的方式进行代码复用；
  2. os.listdir(文件夹路径)，是用列表的形式返回文件夹下的文件名，在构造数据文件路径的时候要用os.path.join()对路径进行拼接；
  3. py中函数return多个对象时，是用元组的形式返回的
  4. 能够使用train_data[idx]的形式访问数据的原因是对于Dataset类型会直接解析为调用Dataset.__getitem__(self,index)函数

训练过程可视化：TensorBoard

TensorBoard作为一个训练的可视化工具，可以帮我们把整个训练过程中产生的一些数据收集起来，形成图像，帮助我们快速直观地了解整个训练过程；

SummaryWriter

SummaryWriter是Pytorch提供的工具类，从而让torch可以结合TensorBoard使用：

  1. 该类可以向日志文件夹中写入event文件，该文件可以被TensorBoard解析，从而形成可视化的结果；
  2. 日志可以通过SummaryWriter的初始化参数指定，如果不指定的话默认创建一个名为runs的文件夹使用

SummaryWriter类常用的方法

  1. add_scalar()
  • 原型：def add_scalar(self, tag, scalar_value, global_step=None, walltime=None)
  • sclar_value，(x,y)中的y，即步-值中的值
  • global_step，(x,y)中的x，即步-值中的步
  • tag(String)，对应图表的标题，通常一个tag会对应一个event文件，即修改tag重新绘制会创建一个新的event文件
  • 如果修改了绘图方式，但是不修改tag，直接运行，两次截然不同的运行结果被写入同一个event文件，tensorboard绘制会出现问题；
  • 官方建议是在有不修改tag，但是要更换绘制方式的需求时，先删除原始的event文件
  1. add_image()
  • 原型：def add_image(self, tag, img_tensor, global_step=None, walltime=None, dataformats='CHW')
  • img_tensor，它对应的就是图像数据，主要有torch.tensor和numpy.array两种类型
  • 图像转换为张量之后通常有高H，宽W，通道C，三个维度，并且通道数C一般是3（rgb）；
  • 该方法默认图像张量是$C\times H\times W$，从参数默认值也可以看出来
  • 如果张量不对应需要手动指定dataformats，例如用PIL的Image打开的图像，由numpy.array()方法转出的图像默认是$H\times W\times C$
  1. close()
  • 在完成写入文件之后使用，关闭IO流

基本使用方法

  1. 创建SummaryWriter对象
  2. 使用相关方法写入日志文件夹中的事件文件
  3. 在命令行使用tensorboard --logdir=日志文件夹名
  • 默认会在本地的6006端口打开一个网页，显示TensorBoard根据事件文件的绘制内容（会绘制所有的事件文件对应的图）；
  • 如果在一个服务器上，多个人都使用tensorboard的话默认端口可能有冲突，可以使用tensorboard --logdir=日志文件夹名 --port==端口号，指定端口；

代码

"""
用来演示如何使用tensorboard记录数据
主要涉及add_scalar和add_image的使用
"""
from torch.utils.tensorboard import SummaryWriter
from PIL import Image
import numpy as np

writer = SummaryWriter("logs")

img_pil1 = Image.open("/Users/yesys/Py_projects/MLpre/DataSets/hymenoptera_data/train/ants/7759525_1363d24e88.jpg")
img_pil2 = Image.open("/Users/yesys/Py_projects/MLpre/DataSets/hymenoptera_data/train/bees/150013791_969d9a968b.jpg")

img_np1 = np.array(img_pil1)
img_np2 = np.array(img_pil2)

writer.add_image("demo_img",img_np1,1,dataformats="HWC")
writer.add_image("demo_img",img_np2,2,dataformats="HWC")

for i in range(100):
    writer.add_scalar("y=2x", 2*i, i)

writer.close()

Transforms

pytorch中用于处理图像的工具（一个py模块），里面有很多好用的工具类

导入方式是：

from torchvision import transforms

ToTensor

这是最常用的类，主要作用是将PIL Image或numpy的对象转换为tensor类型；
（PIL的Image类图像得到Image对象，cv包处理图像得到numpy对象，这是两种常见的图像处理方式）

tensor数据类型是pytorch框架下的一个重要数据类型，几乎所有使用pytorch实现的，涉及模型、网络的相关代码都有这个类型；

关注两个问题：

  1. 为什么要使用tensor数据类型
  2. transform如何使用（以ToTensor为例）

为什么要使用tensor

tensor本身就是pytorch针对神经网络的训练实现的一个数据类型，该类中不仅可以存储与模型、网络相关的数据，还集成了许多训练时所需的属性、方法；

Transform的使用方法

  1. 直接导入其中的工具类，实例化为对象进行使用；
  2. 将其中的某个工具类导入进来，然后创建一个自己的类对其进行继承；即基于一个模板，实现自己针对特定任务所需的趁手工具；
