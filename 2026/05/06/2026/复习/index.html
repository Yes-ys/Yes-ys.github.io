深度学习

梯度下降

  1. 基本原理
  2. BGD
  • 含义
  • 优点：1）梯度精确，下降方向稳定（在优化曲面上，其轨迹最为平滑、稳定） ；2）理论收敛次数较少（线性收敛，SGD是次线性收敛）；3）可充分收敛到极小值，当曲面有唯一极小值或为凸函数时，BGD几乎能够精确地收敛到极小值；4）可以利用并行计算；
  • 缺点：1）每步计算成本较高，需要扫描整个数据集，存在冗余计算（相同的样本）；2）容易卡在鞍点与尖锐极小值点（精确梯度在鞍点处几乎为0，让更新停滞；尖锐极小值点对应泛化能力较差），缺乏跳出能力；
  1. SGD
  • 含义
  • 优点：1）单步极快，实际训练速度快；2）总计算量较小，要达到同样的精度$\epsilon$，SGD的时间复杂度是$O(\epsilon)$，而BGD时间复杂度是$O(N\epsilon)$；3）可以利用噪声逃逸，摆脱鞍点和尖锐局部极小值点；
  • 缺点：1）梯度估计噪声大，损失函数下降过程中存在剧烈波动；2）由于梯度的随机性和固定的学习率，尽管概率论保证其会朝极小值点收敛但是无法像BGD一样精确地完成收敛，最终一般呈现出在极小值附近震荡；3）学习率调节极具挑战，需要精心设计衰减策略；
  1. MiniBatch GD
  • 含义
  • 优点：1）计算效率相比BGD更高，可以并行化实现，是可接受的；2）梯度估计相较于SGD噪声小很多，更新方向更稳定，减小了震荡容易逼近精确值；3）相较于BGD梯度估计仍然带有部分噪声，具有逃逸能力；
  • 缺点：1）batch_size、学习率的超参数需要精心设计；2）仍然属于SGD、BGD之间的一种权衡，噪声的存在具有两面性，可能无法达到最优；
  • 选取batch_size与epoch的关系：batch_size增加，epoch一般也应该增加；
  • 原因：1）补偿减小的更新步数（1个epoch的更新次数 = 样本总数/batch_size）；2）补偿减小的探索能力（batch_size增大对应的噪声减小，跳出鞍点、尖锐极小值的能力减弱，用更多的epoch来补偿）；3）配合大学习率衰减（batch_size增大一般可以等比例大增大学习率，然后采用学习率衰减的策略，长epoch有利于学习率衰减）
  1. 动量随机梯度下降
  • 含义：指数加权移动平均
  • 优点

自适应优化器

  1. Adagrad
  • 特点：1）每个参数有自己的学习率；2）用s累计参数对应梯度中的更新量的平方；3）更新时学习率除以平方；
  • 优点
  • 缺点
  1. RMSprop
  • 特点：在Adagrad基础上，将s更新改为指数加权移动平均，从而减小遥远梯度的影响
  1. Adam
  • 特点：1）RMSprop+Momentum，采用了RMSprop相同的逐元素的学习率缩放策略，此外更新时的梯度值使用动量随机梯度下降得到，对应的值分别称为一阶矩$m_t$和二阶矩$v_t$；对应指数加权移动平均的衰减率为$\beta_1,\beta_2$；2）再实际使用之前要用$m_t/(1-\beta^t_1),v_t/(1-\beta^t_2)$进行偏差修正；因为训练初期一、二阶矩偏向0会导致模型参数发生巨大的跳动，后期修正分母会逐渐趋于1；

Batch Normalization

  1. 操作过程 线性变换之后 激活函数之前
  2. 解决的问题：内部协变量偏移
  • 协变量偏移：机器学习中训练集和测试集的输入分布不一致，导致模型泛化困难。
  • 内部协变量偏移：对于网络的任意一层，它接收的是前一层输出的特征（即它的“输入”）。当前面层的参数被梯度更新后，特征输出就会改变——这意味着该层的输入分布一直在变。
  1. 目的：为了让网络在每一层都保持相近的分布
  2. 深入理解
  • 正规化手段 强行分布 均值0 方差1
  • 使得激活函数输入值处在合理区间 防止梯度爆炸或梯度消失（对于sigmoid激活函数，如果输入落在远离0的饱和区，该项导函数值非常小，会导致该激活函数以前的参数在计算对应梯度值时变得非常小，发生梯度消失）

为什么BN层一般用在线性层和卷积层之后，而不是放在非线性单元之后

  1. BN设计的初衷是为了主要是为了控制激活函数输入的分布，放在后面无法控制该输入，输入仍然会跑到饱和区导致梯度消失之类的问题；
  2. BN 的标准化操作（减均值、除标准差）针对的是近似对称、无界的分布效果最好。线性层输出根据中心极限定理近似高斯分布，对它们进行归一化会产生一个更加稳定的分布

BN层计算均值的时候计算多少次

  1. 假设[b,c,h,w]计算bhw次，针对每个通道有一层BN
  2. BN层有2c个参数（归一化后带一个缩放、平移的参数，$\gamma,\beta$）

监督学习——生成式 v.s. 判别式

  1. 生成式模型：利用贝叶斯公式对联合概率（类条件概率、先验概率）建模，最终得到某类数据的分布模型，间接地再得到后验概率，用于分类；
  • 典型代表 1 2 3 4 5
  • 优点 在模型假设数据分布与真实分布接近的情况下收敛较快，可以用于生成数据
  • 缺点 计算资源量大
  1. 判别式模型：直接尝试对后验概率建模，不关心数据的实际分布，而是尝试学习到不同类别数据的边界条件，从而实现分类；
  • 典型代表：1 2 3 4 5
  • 优点 计算资源量大
  • 缺点 不能生成数据
  1. 关系
  • 都是为了得到后验概率
  • 生成式->判别式

L1正则化与L2正则化的区别与联系

  1. 联系：都是施加惩罚项，迫使模型使用更小的权重，从而降低方差

Dropout

  • Dropout层实现的方式（作用于前一层神经元的输出，伯努利分布采样0-1 mask矩阵，0对应丢弃上一层某个神经元；结果要除以1-p防止输出期望被系统性缩小(1-p)倍）

理解指定维度的操作很重要

切片语法

切片语法的作用：高效地在原始张量的基础上提取出所需 维度/索引 的数据

按照最一般的情况（5维张量）对应(批次，时间步，通道，高度，宽度)

x = torch.rand(2,5,3,16,16) #2个批次 5个时间步 3个通道 16*16的数据

常用符号之“:”

  1. 全选某维度的元素
    x_view = x[:,:,:,:,:] # 注意点1：x_view 与 x是共享内存的（切片操作默认得到的是视图，对应原始5维张量）
x_view = x[1,1,:,:,:] # 注意点2：整数索引会带来“降维”，这里得到的是torch.Size([3,16,16])的张量视图（对应x张量在第1批次、第1个时间步的样本）
  2. 选择某维度指定范围的元素（左闭右开）
    x_view = x[1,3:5,:,:,:] # 注意点3：时间步选取范围左闭右开对应实际索引是3~4，因此得到的张量形状是torch.Size([2,3,16,16])
x_view = x[1,:5,:,:,:] # 注意点4：如果指定范围前面省略默认从0开始，得到张量形状是torch.Size([5,3,16,16])
  3. 添加维度
    x_view = x[:,:,:,:,:,None] # 注意点5：等价于unsqueeze(5)，在最后插入一个新维度（索引从0开始）当然None也可以出现在起始维度（等价于unsequeeze(0)） 或者出现在中间 即使插入了新维度，默认得到的也是视图
  4. 步长采样
    x_view = x[:,::2,:,:,:] # 注意点6：在时间步维度上每2个索引采样一步，省略开始索引默认从0开始，因此采样到的是0 2 4，对应torch.Size([2,3,3,16,16])
x_view = x[:,1::2,:,:,:]# 注意点7：指定开始采样维度，采样得到的是1 3，对应torch.Size([2,2,3,16,16])
  5. 反向操作
    x_view = x[:,::-1,:,:,:] # 注意点8：“-”代表反向操作，反向操作时默认从“末尾索引”开始，到“开始索引”（0）结束
x_view = x[1,3:5:-1,:,:,:] # 注意点9：“-”可以搭配前面的多种操作使用，例如此处是对时间步维度选择指定范围之后再反向
  6. 省略操作
    x_view = x[...,:4,:4] # 注意点10：“...”代表范围内所有维度的所有位置都被选中，例子中操作的含义是保留所有批次、时间步、通道中样本的左上角4*4的部分

高级索引

整数张量/整数列表索引

  1. 使用torch.Tensor/NumPy数组
  2. 索引取得张量的维度是：高级索引数组（广播后，若有）的维数 + 未被高级索引覆盖的维数（这些维数由切片、省略号等保留）
  • 注意广播后的高级索引维度会被“提到最前面”，而不是插在原位置

eg1 没有广播的情况

import torch

x = torch.arange(24).reshape(2, 3, 4)  # shape (2, 3, 4)
i = torch.tensor([[0,1], [1,0]])        # shape (2,2)
j = torch.tensor([[1,2], [2,1]])        # shape (2,2)

# 索引：x[i, j]   # 第0轴用 i，第1轴用 j，第2轴不写（默认:）
# 实际等价于 x[i, j, :] 结果形状——广播：i 与 j 形状都是 (2,2)，广播后还是 (2,2)；保留轴：第三轴长度 4；结果形状 = (2,2) + (4,) = (2,2,4)

out = x[i, j]  # 实际等价于 x[i, j, :]
print(out.shape)  # torch.Size([2, 2, 4])

eg2 有广播的情况

x = torch.randn(5, 7, 3)        # (5, 7, 3)
idx1 = torch.tensor([0, 1]).reshape(2, 1)   # (2, 1)
idx2 = torch.tensor([0, 1, 2]).reshape(1, 3) # (1, 3)

out = x[idx1, :, idx2]          # 第0、2轴用高级索引，第1轴切片
# 广播 idx1(2,1) 与 idx2(1,3) → (2,3)
# 保留轴：第1轴长度 7
# 结果形状：(2, 3) + (7,) = (2, 3, 7) 高级索引的轴保留相对顺序，提到最前面来
print(out.shape)  # torch.Size([2, 3, 7])

eg3 单一维度上使用高级索引

x = torch.randn(5, 7, 3)        # (5, 7, 3)
idx = torch.tensor([[0,3],[2,4]]) # (2,2)

out = x[idx]                     # 只索引第0轴，其余轴全保留
# 广播形状：(2,2)
# 保留轴：第1轴 7，第2轴 3
# 结果形状：(2,2) + (7,3) = (2,2,7,3)
print(out.shape)  # torch.Size([2, 2, 7, 3])

上述维度变换的关系是：

  1. x的第0维会按照idx变成(2,2)的形状
  2. 对应有4个元素
  • out[0,0]<->idx[0,0]<->”0”，按照0索引到x[0]，即out[0,0]处为x[0]处的(7,3)形状的矩阵
  • out[0,1]<->idx[0,1]<->”3”，按照1索引到x[3]
  • out[1,0]<->idx[1,0]<->”2”<->x[2]
  • out[1,1]<->idx[1,1]<->”4”<->x[4]

注意，按照高级索引选取维度的规则，甚至存在这种让维度变多的情况
注意，与切片不同，一旦使用了高级索引得到的是副本而不是视图
高级索引可以和切片结合使用

补充：分支界限算法

  1. 思想：用于求解最优化问题，维护一个 “活结点表” ，起初只有根结点。算法不断从表中取出一个结点（成为当前扩展结点），生成它的所有孩子结点，再通过 约束函数 和 限界函数 检查哪些孩子结点可能包含最优。只有通过检查的孩子才会被插入活结点表，等待后续扩展。当活结点表为空时，算法结束，当前最优解就是全局最优解
  • 约束函数：排除不可行解（如背包超重）。
  • 限界函数：排除那些即使继续搜索也无法得到优于当前最优解的结点。
  1. 例子：0-1背包问题
  • 状态节点来自于对当前物品是否选择
  • 约束函数即节点扩展后当前背包是否超重
  • 界限函数（上界值）可以用分数背包估计：1）将物品按照价值重量比降序排列，扩展一个节点后针对剩余空间从当前可选物品开始按照分数背包的规则贪心拿取，填满背包时得到的就是上界（界限函数），如果该值小于当前最优值则该节点不能纳入活节点表（剪枝）
    （本质上就是暴力搜索加了一些剪枝）

微信撤回机制

  1. 客户端发送撤回请求，包含消息ID、时间戳、会话ID等信息。
  2. 服务器验证该请求是否合法（如是否在撤回时间窗口内、用户是否有权限）。
  3. 服务器将撤回指令广播给所有接收该消息的客户端。
  4. 各客户端接收到指令后，根据消息ID删除或替换本地消息。
