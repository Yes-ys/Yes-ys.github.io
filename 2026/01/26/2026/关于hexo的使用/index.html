关于hexo的使用

这篇blog用来记录关于hexo的使用，因为虽然我已经写了2年博客了，一直在用hexo d、hexo g、hexo new p...，博客目录管理目前也是一片混乱

整个blog项目结构

这部分是ai写的，我自己改了一点点，总之就是方便自己熟悉博客项目的结构

  • _config.yml: 站点的主配置文件，这里配置了网站标题、作者、语言、URL、永久链接格式以及主题选择等核心信息。
  • _config.butterfly.yml: Butterfly 主题的特定配置文件（通常用于覆盖 themes/butterfly/_config.yml 中的默认配置，方便升级主题时不丢失配置）。
  • package.json: NodeJS 项目的标准文件，记录了 Hexo 框架、插件以及主题所需的各种依赖包版本信息。
  • scaffolds/: 模版文件夹（脚手架）。当你执行 hexo new <layout> <title> 时，Hexo 会根据这里对应的模版文件（如 post.md, page.md, draft.md）来创建新文件。其中post是用来编写文章的模板最常用的，draft是草稿，目前没用过。page可以用来自定义页面，常用的有主页面、分类页面、介绍页面（目前我有的也是这些）；
  • source/: 资源文件夹，这是你平时主要操作的地方。
    • _posts/: 存放你撰写的所有 Markdown 博客文章。现在已经按照年份文件夹（如 2025/, 2026/）进行了分类。
    • about/, tags/, archives/: 这些目录对应博客的独立页面（Page），通常包含 index.md。
  • themes/: 主题文件夹。下载的主题（如 butterfly, PreciousJoy）源码存放于此。
  • public/: 静态文件输出目录。执行 hexo g 后，Hexo 会将 Markdown 和主题模板编译成 HTML/CSS/JS 放在这里，hexo d 部署的就是这个目录的内容。
  • db.json: Hexo 的缓存文件，存储了站点的数据模型（文章、分类、标签等），用于加速生成过程。
  • organize_posts.py: 自定义的 Python 脚本，用于按年份整理 _posts 下的文章。
  • source_image/: 自定义文件夹，用于存放一些非文章特定的图片资源。
