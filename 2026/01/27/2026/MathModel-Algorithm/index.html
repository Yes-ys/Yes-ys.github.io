归一化处理

问题：对不同量纲的数据归一化处理

简单数据：

  • 身高: [160, 175, 182, 155, 168] (cm)
  • 体重: [55, 70, 85, 50, 65] (kg)
  • 成绩: [78, 92, 85, 68, 88] (分)

为什么要归一化处理？

  1. 比较直接的理解：消除量纲影响，实现公平比较：身高的单位是厘米（通常数值在150-200之间），体重的单位是公斤（通常数值在50-100之间），成绩的单位是分（在0-100之间）。数字的绝对大小和尺度完全不同。如果不处理，在后续的数学模型（如计算距离、求均值、主成分分析等）中，数值范围大的特征（如身高）会“淹没”数值范围小的特征（如成绩），导致结果失真。
  2. 其他应用：加速模型收敛：在机器学习的梯度下降等优化算法中，特征尺度差异大会导致损失函数的“等高线”呈扁长的椭圆状，优化路径会曲折缓慢。归一化后，“等高线”更接近圆形，算法能更快、更稳定地找到最优解。
  3. 其他应用：提高模型精度与稳定性：许多模型（如K-近邻、支持向量机、逻辑回归、神经网络）的核心计算都依赖于特征之间的距离或内积。如果某个特征数值巨大，它将主导整个距离或内积的计算，使模型无法有效学习其他特征的信息，从而降低精度和泛化能力。

分类变量编码

问题: 学生信息中的分类变量处理

数据：

  • 学生ID: [1, 2, 3, 4, 5]
  • 专业: [“数学”, “物理”, “数学”, “化学”, “物理”]
  • 年级: [“大一”, “大二”, “大三”, “大一”, “大二”]

为什么要进行编码？

机器学习模型（如逻辑回归、决策树、神经网络等）只能处理数值数据。因此，我们必须将文本类别的“专业”和“年级”转换为数值。但是，不同的转换方法有不同的含义和适用场景。本质是为样本手动引入特征属性，并赋予相应的值

针对上述例子：分类变量如“专业”和“年级”是文本类型，不能直接用于大多数数学和机器学习模型，因为这些模型需要数值输入。因此，我们需要将这些分类变量转换为数值形式。常见的编码方法有标签编码（Label Encoding）和独热编码（One-Hot Encoding）。

连续变量离散化

问题: 年龄分组处理，将下面几个年龄的样本划分为几组；

数据：

  • 年龄: [18, 22, 25, 30, 35, 42, 48, 52, 60, 65, 70]

为什要进行离散化？

连续变量离散化（分箱）是将连续数值转换为有限个离散区间的过程。这种转换在数据分析和机器学习中有许多应用：1）非线性关系建模，许多现实世界的关系是非线性的，离散化可以捕捉这些模式；2）提高可解释性，比起连续数据的变化趋势，人类更容易理解分类组的特点；3）算法要求，某些算法（如决策树、朴素贝叶斯）天然更适合离散变量

这里本质上是对具有连续特征属性的样本进行了分类，但是如何真正的将连续的特征值转换为离散的特征值？

或许可以在完成分类之后，又对类别进行编码；

  • 选手: A, B, C, D, E, F

但实际上相关性分析更有价值的用法在于论证建模的可解释性，这意味着我们通常会把相关性分析和其他方法相互结合

  1. 和数据降维结合：在进行数据降维之前，我们可以用相关性分析先论证当前样本的各个特征变量之间存在较强的关联，此时意味着增加的数据维度并不会增加对于结果刻画能力，因为从某种角度来讲可能存在多个维度实际上反应的是一个特征，例如“年龄大小、是否喜欢电子游戏”（从某个方面来讲）…
  2. 避免多重共线性：本质上还是数据降低维，此时两种特征变量的关系更强（几乎存在线性关系）；
  3. 论证先验特征与目标变量之间存在可解释的统计关系：例如在使用日期（年-月-日）来预测某地区的降雨量，那么在考虑回归模型之前我们应该先论证二者有关（*例如，在北半球夏季的降雨量肯定是较多的，但是并非所有事物都有这么明显的关系…*）
  4. 论证一些变量不重要，抛弃它们；

评价类问题

评价类问题的核心目标是：根据一定的标准，对一组（多个）对象的优劣、等级、水平或风险进行系统性的分析、比较和排序。

常见应用实例：

  1. “最佳选址”问题：评价几个候选地点的优劣（指标：地价、交通、人口、环境等）。
  2. “企业竞争力”评价：对多家公司进行排名（指标：营收、利润率、创新能力、市场份额等）。
  3. “生态环境质量”评估：评价不同区域的环境状况（指标：PM2.5浓度、水质指数、绿地覆盖率等）。
  4. “风险预警”问题：评价不同金融机构或地区的风险等级。
  5. “奖学金评定”：评价学生的综合表现（指标：成绩、科研成果、社会实践等）。

常用算法速览

  1. 层次分析法：无数据支撑下主要是给指标设置权重；
  2. 权熵法：有数据支撑下给指标设置权重；
  3. 灰色关联分析：分析各种因素对结果的影响程度、解决与时间变化有关的综合评价问题；
  4. TOPSIS模型：对评价结果进行排序，或评价的准则层太多，或准则层中的指标相对权重已知（不能用层次分析法的情况）；
  5. 模糊综合评价算法：模糊环境下，考虑多因素，为了某种目的对事物做出综合决策的方法；
  6. 神经网络算法：指标较多，有训练数据支撑，并且需要对未知数据进行评价；
  7. 数据包络法（DEA）：多种投入和多种产出的评价问题；
  8. 秩和比综合方法：评价多个指标的综合水平情况，医学领域应用广泛；

从评价问题开始主要涉及解决具体问题的方法使用，下面会简单介绍一下相关方法针对性的问题，方便能够再拿到一个问题的时候快速联想到

层次分析法

思想：在没有数据支撑的情况下，主观的进行一些“两两对比”的打分，最终能够评出最好的对象；

一些细节：（以决定买哪个手机为例子，三个手机X、Y、Z，3个指标，性能、外观、价格）

  1. 主要分为三层，目标层、准则层、方案层
  2. 构建各个层次的决策矩阵，目标层对准则层的决策矩阵会确定各个准则的权重（对应就是性能、外观、价格在参与评价时的权重）；
  3. 准则层对方案层的决策矩阵，会确定每款手机在每个准则上的优劣（矩阵中会应用刚刚的权重）；
  4. 合成总的权重，对应每款手机的最终得分，得分最高的手机就是最好的手机；

注意：2、3中每一次构建决策矩阵后都需要进行一致性检验，避免出现主观判断“A>B， B>C，但C>A”的逻辑矛盾。必须检验矩阵的一致性是否可接受。

优化类问题

常用算法速览

  1. 线性规划模型：目标函数和约束条件均为线性；
  2. 整数规划或0-1规划：决策变量取值被限定为整数或0、1；
  3. 动态优化模型：以时间为划分阶段的动态过程优化；
  4. 非线性规划模型：目标函数或者约束中含有非线性函数；
  5. 多目标规划模型：目标函数不唯一，存在多个优化目标；
  6. 最速下降、随机梯度下降、拟牛顿法：目标函数为凸函数；
  7. 粒子群算法、模拟退火、遗传算法：目标函数为非凸函数，智能优化算法，依次针对决策变量为连续变量、决策变量离散或连续（维度较低）、决策变量为离散变量；
