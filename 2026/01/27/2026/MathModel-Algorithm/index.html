MathModel-Algorithm

这篇blog用来记录数学建模中常用的算法，针对不同问题进行叙述

数据预处理

常用算法速览

  1. 拉格朗日插值法和牛顿插值法：补足数据中的缺失值；
  2. 正态分布$3\sigma$原则、箱型图：检测数据中的异常值；
  3. 标准差法、极值差法、功效系数法：对数据进行归一化处理；
  4. 独热编码、标签编码：数据中存在分类变量；
  5. 等宽法、等频法、聚类方法：连续变量离散化；
  6. PCA主成分分析、T-SNE降维算法、UMAP降维法：数据变量维度过高，需要对数据进行降维处理；

归一化处理

问题：对不同量纲的数据归一化处理

简单数据：

  • 身高: [160, 175, 182, 155, 168] (cm)
  • 体重: [55, 70, 85, 50, 65] (kg)
  • 成绩: [78, 92, 85, 68, 88] (分)

为什么要归一化处理？

  1. 比较直接的理解：消除量纲影响，实现公平比较：身高的单位是厘米（通常数值在150-200之间），体重的单位是公斤（通常数值在50-100之间），成绩的单位是分（在0-100之间）。数字的绝对大小和尺度完全不同。如果不处理，在后续的数学模型（如计算距离、求均值、主成分分析等）中，数值范围大的特征（如身高）会“淹没”数值范围小的特征（如成绩），导致结果失真。
  2. 其他应用：加速模型收敛：在机器学习的梯度下降等优化算法中，特征尺度差异大会导致损失函数的“等高线”呈扁长的椭圆状，优化路径会曲折缓慢。归一化后，“等高线”更接近圆形，算法能更快、更稳定地找到最优解。
  3. 其他应用：提高模型精度与稳定性：许多模型（如K-近邻、支持向量机、逻辑回归、神经网络）的核心计算都依赖于特征之间的距离或内积。如果某个特征数值巨大，它将主导整个距离或内积的计算，使模型无法有效学习其他特征的信息，从而降低精度和泛化能力。

分类变量编码

问题: 学生信息中的分类变量处理

数据：

  • 学生ID: [1, 2, 3, 4, 5]
  • 专业: [“数学”, “物理”, “数学”, “化学”, “物理”]
  • 年级: [“大一”, “大二”, “大三”, “大一”, “大二”]

为什么要进行编码？

机器学习模型（如逻辑回归、决策树、神经网络等）只能处理数值数据。因此，我们必须将文本类别的“专业”和“年级”转换为数值。但是，不同的转换方法有不同的含义和适用场景。本质是为样本手动引入特征属性，并赋予相应的值

针对上述例子：分类变量如“专业”和“年级”是文本类型，不能直接用于大多数数学和机器学习模型，因为这些模型需要数值输入。因此，我们需要将这些分类变量转换为数值形式。常见的编码方法有标签编码（Label Encoding）和独热编码（One-Hot Encoding）。

连续变量离散化

问题: 年龄分组处理，将下面几个年龄的样本划分为几组；

数据：

  • 年龄: [18, 22, 25, 30, 35, 42, 48, 52, 60, 65, 70]

为什要进行离散化？

连续变量离散化（分箱）是将连续数值转换为有限个离散区间的过程。这种转换在数据分析和机器学习中有许多应用：1）非线性关系建模，许多现实世界的关系是非线性的，离散化可以捕捉这些模式；2）提高可解释性，比起连续数据的变化趋势，人类更容易理解分类组的特点；3）算法要求，某些算法（如决策树、朴素贝叶斯）天然更适合离散变量

这里本质上是对具有连续特征属性的样本进行了分类，但是如何真正的将连续的特征值转换为离散的特征值？

或许可以在完成分类之后，又对类别进行编码；

kendall相关系数应用

问题：两位评委对6位选手的排名是否呈现了“一致性”

数据：

  • 选手: A, B, C, D, E, F
  • 评委1排名: [1, 2, 3, 4, 5, 6]
  • 评委2排名: [2, 1, 4, 3, 6, 5]

但实际上相关性分析更有价值的用法在于论证建模的可解释性，这意味着我们通常会把相关性分析和其他方法相互结合

  1. 和数据降维结合：在进行数据降维之前，我们可以用相关性分析先论证当前样本的各个特征变量之间存在较强的关联，此时意味着增加的数据维度并不会增加对于结果刻画能力，因为从某种角度来讲可能存在多个维度实际上反应的是一个特征，例如“年龄大小、是否喜欢电子游戏”（从某个方面来讲）…
  2. 避免多重共线性：本质上还是数据降低维，此时两种特征变量的关系更强（几乎存在线性关系）；
  3. 论证先验特征与目标变量之间存在可解释的统计关系：例如在使用日期（年-月-日）来预测某地区的降雨量，那么在考虑回归模型之前我们应该先论证二者有关（*例如，在北半球夏季的降雨量肯定是较多的，但是并非所有事物都有这么明显的关系…*）
  4. 论证一些变量不重要，抛弃它们；

评价类问题

评价类问题的核心目标是：根据一定的标准，对一组（多个）对象的优劣、等级、水平或风险进行系统性的分析、比较和排序。

常见应用实例：

  1. “最佳选址”问题：评价几个候选地点的优劣（指标：地价、交通、人口、环境等）。
  2. “企业竞争力”评价：对多家公司进行排名（指标：营收、利润率、创新能力、市场份额等）。
  3. “生态环境质量”评估：评价不同区域的环境状况（指标：PM2.5浓度、水质指数、绿地覆盖率等）。
  4. “风险预警”问题：评价不同金融机构或地区的风险等级。
  5. “奖学金评定”：评价学生的综合表现（指标：成绩、科研成果、社会实践等）。

常用算法速览

  1. 层次分析法：无数据支撑下主要是给指标设置权重；
  2. 权熵法：有数据支撑下给指标设置权重；
  3. 灰色关联分析：分析各种因素对结果的影响程度、解决与时间变化有关的综合评价问题；
  4. TOPSIS模型：对评价结果进行排序，或评价的准则层太多，或准则层中的指标相对权重已知（不能用层次分析法的情况）；
  5. 模糊综合评价算法：模糊环境下，考虑多因素，为了某种目的对事物做出综合决策的方法；
  6. 神经网络算法：指标较多，有训练数据支撑，并且需要对未知数据进行评价；
  7. 数据包络法（DEA）：多种投入和多种产出的评价问题；
  8. 秩和比综合方法：评价多个指标的综合水平情况，医学领域应用广泛；

相关应用

从评价问题开始主要涉及解决具体问题的方法使用，下面会简单介绍一下相关方法针对性的问题，方便能够再拿到一个问题的时候快速联想到

层次分析法

思想：在没有数据支撑的情况下，主观的进行一些“两两对比”的打分，最终能够评出最好的对象；

一些细节：（以决定买哪个手机为例子，三个手机X、Y、Z，3个指标，性能、外观、价格）

  1. 主要分为三层，目标层、准则层、方案层
  2. 构建各个层次的决策矩阵，目标层对准则层的决策矩阵会确定各个准则的权重（对应就是性能、外观、价格在参与评价时的权重）；
  3. 准则层对方案层的决策矩阵，会确定每款手机在每个准则上的优劣（矩阵中会应用刚刚的权重）；
  4. 合成总的权重，对应每款手机的最终得分，得分最高的手机就是最好的手机；

权熵法

思想：完全由数据驱动。数据之间的差异越大（方差越大），包含的信息量就越大，对应的权重就越高。这是一种客观赋权法，避免了人为因素的干扰。

简单数据与分析：

  • 比如评价两个学生的综合素质，指标是【数学成绩，体育成绩】。
  • 学生A：[98, 90]，学生B：[98, 60]。
  • 在“数学”上两人没差距，说明这个指标在当前评价体系下区分度低，权重应该低。
  • 在“体育”上差距很大，说明这个指标区分度高，权重应该高。

TOPSIS (优劣解距离法)

思想：不求完美，只求最不烂。构造一个“理想最优解”和一个“理想最劣解”，看哪个方案离最优解最近，同时离最劣解最远。

例子：选购笔记本电脑（价格最少、性能最高）。

  • 理想最优解：价格0元，性能无限高（现实不存在，是虚拟的）。
  • 理想最劣解：价格无限高，性能为0。
  • 计算备选电脑A、B、C与这两个点的几何距离，综合得分进行排序。

灰色关联分析

思想：看曲线像不像。对于样本量极少（比如只有几年的数据）、信息不完全的“灰色”系统，通过分析两个序列曲线的几何形状相似程度来判断联系是否紧密。

例子：分析哪些因素对“某市GDP”影响最大。

  • 母序列：某市历年GDP。
  • 子序列：历年工业产值、历年农业产值、历年服务业产值。
  • 看哪个子序列的曲线走势与GDP曲线最贴合（关联度最高）。

模糊综合评价

思想：量化“感觉”。处理“很好、较好、一般、差”这种模糊语言描述。利用隶属度函数将定性描述转化为定量评分。

例子：对企业员工进行年度考核。

  • 既然无法精确打分，就让部门主管、同事给出评价等级（如：30%认为优秀，50%认为良好，20%认为一般）。
  • 结合各指标权重，通过矩阵运算计算出该员工最终归属于哪个等级。

数据包络分析 (DEA)

思想：评价“性价比”/效率。在多投入、多产出的情况下，判断决策单元是否有效。

例子：评价10家银行分行的运营效率。

  • 投入指标：员工人数、运营成本、柜台数。
  • 产出指标：存款总额、贷款总额、新增客户数。
  • 不需要预设权重，DEA会自动判断哪些分行在现有投入下产出是相对最优的（位于生产前沿面上）。

预测类问题

常用算法速览

  1. Logistic预测模型（单调递增时间序列的数据）
  2. 灰色预测模型（单调递增时间序列的数据）
  3. 二次指数平滑预测（单调递增时间序列的数据、不规律的时间序列的数据）
  4. ARMA时间序列预测（单调递增时间序列的数据、不规律的时间序列的数据、周期性时间数据）
  5. 季节指数预测模型（单调递增时间序列的数据、不规律的时间序列的数据、周期性时间数据）
  6. BP神经网络预测模型（单调递增时间序列的数据、周期性时间数据、多个指标的时间序列预测）
  7. 高斯回归预测模型（不规律的时间序列的数据）
  8. 马尔可夫预测模型（某系统在已知条件下，系统未来时刻只与当前有关而与过去无关）
  9. 回归分析预测模型（自变量和因变量之间有逻辑相关性）

相关应用

时间序列分析 (ARMA / 指数平滑)

思想：惯性原理。承认事物的发展是可以延续的，利用过去的数据推测未来。其中ARMA关注数据的自相关性，指数平滑关注近期数据权重。

例子：预测某支股票下周的价格波动；根据过去12个月的销售记录预测下个月的销量。

灰色预测 (GM(1,1))

思想：贫信息挖掘。适合数据量极少（如只有4-10个数据）且呈现指数增长趋势的系统。通过对原始数据进行累加生成，弱化随机干扰，由无规律变有规律。

例子：预测某个新出现的传染病的早期确诊人数（初期数据很少）；预测某新产品的早期市场占有率。

马尔可夫预测

思想：无后效性。“明天”的状态只与“今天”有关，而与“昨天”无关。通过统计状态间的转移概率矩阵来预测长期趋势。

例子：

  • 市场占有率预测：已知今年A品牌用户有20%明年会换B品牌，B品牌用户有10%会换A品牌，预测若干年后的市场格局。
  • 天气预测：已知“晴转阴”、“阴转雨”的概率，预测未来的天气序列。

回归分析预测

思想：因果推断。假设因变量Y与自变量X之间存在某种函数关系（线性或非线性），利用最小二乘法求出参数。

例子：根据“广告投入费用”和“促销力度”来预测“产品销量”。

优化类问题

常用算法速览

  1. 线性规划模型：目标函数和约束条件均为线性；
  2. 整数规划或0-1规划：决策变量取值被限定为整数或0、1；
  3. 动态优化模型：以时间为划分阶段的动态过程优化；
  4. 非线性规划模型：目标函数或者约束中含有非线性函数；
  5. 多目标规划模型：目标函数不唯一，存在多个优化目标；
  6. 最速下降、随机梯度下降、拟牛顿法：目标函数为凸函数；
  7. 粒子群算法、模拟退火、遗传算法：目标函数为非凸函数，智能优化算法，依次针对决策变量为连续变量、决策变量离散或连续（维度较低）、决策变量为离散变量；

相关应用

线性规划 / 整数规划

思想：资源配置最大化。在有限的资源（约束条件）下，如何分配使得效益（目标函数）最大。

例子：

  • 生产调度：工厂有A、B两种机床，加工X、Y两种零件，耗时不同，利润不同，机床工作时长有限，问各生产多少利润最大？
  • 选址问题（0-1规划）：在10个备选点中建立3个仓库，覆盖所有需求点且成本最低。

智能优化算法 (遗传算法 / 粒子群 / 模拟退火)

思想：仿生/物理模拟。当问题极其复杂（非线性、多峰值、不可导）无法用数学公式直接求解时，模拟自然界的进化、鸟群觅食或物理退火过程，在解空间中随机搜索并逼近最优解。

例子：

  • 旅行商问题 (TSP)：一个快递员要送30个点，如何规划路线使得总路程最短？（传统的排列组合数是天文数字，必须用智能算法）。
  • 复杂工程系统的参数整定：比如寻找神经网络的最优超参数。

无监督聚类

  2. 层次聚类
  3. 高斯混合聚类
  4. SOM自组织神经网络

相关应用

K-means 聚类 (无监督)

思想：物以类聚。预先设定要分K类，随机选取中心，计算距离并不断迭代调整中心，直到分类稳定。

例子：

  • 客户细分：电商平台根据用户的“消费金额”和“消费频次”，自动将用户划分为“高价值客户”、“挽留客户”、“一般客户”，以便精准营销。

KNN / 决策树 (监督分类)

思想：经验学习。

  • KNN：近朱者赤。新样本离谁最近，就属于谁那类。
  • 决策树：像人类做决策一样，通过一系列“是/否”的问题（特征判断）将样本逐层筛选分类。

例子：

  • 信用评级：银行根据申请人的年龄、收入、是否有房等特征（已知历史数据的标签），判断新申请人的违约风险是“高”还是“低”。
  • 垃圾邮件识别：根据邮件中的关键词频率，判断是否为垃圾邮件。

图与网络

常用算法速览

  1. Dijkstra
  2. Floyd
  3. 图+规划模型：TSP问题

相关应用

最短路径 (Dijkstra / Floyd)

思想：贪心/动态规划。在加权图中寻找两点间权值之和最小的路径。

例子：

  • 导航软件：规划从北京到上海的最短驾车路线（权值可以是距离，也可以是时间）。
  • 网络路由：数据包在网络节点间传输的最佳路径选择。

最小生成树 (Prim / Kruskal) & TSP

思想：连通性与遍历。

  • 最小生成树：用最低成本连接所有节点。
  • TSP：以最低成本遍历所有节点并回到起点。

例子：

  • 光缆铺设：在N个城市间铺设光缆，如何连接能使总造价最低且所有城市互通？（最小生成树）
  • 物流配送路径规划（TSP）。

微分模型

常用算法速览

  1. 人口模型
  2. 战争模型
  3. 传染病模型

相关应用

传染病模型 (SI / SIR / SEIR)

思想：动力学流转。将人群划分为不同的仓室（如：易感者S、感染者I、移出者R），利用微分方程组描述不同人群之间随时间的转化速率（如感染率、康复率）。

例子：

  • 预测流感或COVID-19的疫情发展趋势：预测确诊人数何时达到峰值？峰值是多少？
  • 评估干预措施：如果实施隔离（降低感染率），疫情曲线会如何平缓？

人口增长模型 (Malthus / Logistic)

思想：增长与阻滞。

  • Malthus：理想状态下的指数增长。
  • Logistic：考虑环境容纳量（资源限制），增长率随人口增加而下降，呈S型曲线。

例子：预测某国家未来的人口数量；预测某种微生物在培养皿中的生长数量。
