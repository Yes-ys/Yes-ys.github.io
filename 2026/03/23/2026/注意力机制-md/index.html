注意力机制

注意力提示（相关基本概念）

注意力的双组件框架

  1. 非自主性提示，来自吸引你注意力的物品本身客观自带的突出的特点；
  2. 自主性提示，来自你的大脑对你当前注意力的控制；

双组件的神经网络表示

  1. 非自主性提示
  • 例如要想将选择偏向于感官输入，则可以简单地使用参数化的全连接层，甚至是非参数化的最大汇聚层或平均汇聚层。
  • 当然，这称不上注意力，所以实际上只有自主性提示才称得上我们在深度学习中提到的“注意力”
  1. 自主性提示（在注意力机制背景下，称为查询），它作为“神经网络”的工作流程大致如下
  • 1）注意力机制通过注意力汇聚（attention pooling）将选择引导至感官输入（sensory inputs，例如中间特征表示）
  • 2）在注意力机制中，这些感官输入被称为值（value）。 更通俗的解释，每个值都与一个键（key）配对， 这可以想象为感官输入的非自主提示。

Nadaraya-Watson核回归

接下来一步一步看如何实现一个最简单的带注意力机制的模型

假设我们有数据集${x_i,y_i}$

我们要训练一个模型来根据输入的$x_i$预测输出的$\hat{y}_i$

先考虑两个不用训练的非参数模型：

  1. 平均汇聚
  • $\hat{y} = \frac{1}{n}\sum_i^n y_i$
  • 这种方式所有预测输出都是现有数据输出的平均值
  • 缺点是没有利用输入$x$的信息
  1. Nadaraya-Watson核回归
  • $\hat{y} = \sum^n_{i=1}\frac{K(x-x_i)}{\sum^n_{j=1}K(x-x_j)}y_i$
  • K是核函数，这里其具备的性质是当输入绝对值越接近0时越大
  • 显然该非参数模型中，分母是确定的，撇开分母就是一个加权求和，而权重则来自核函数；并且根据上述核函数的性质，此时$x$与$x_i$越接近，加权时给到$y_i$的权重就越大
  • 类似于：$x$是查询，$x_i$是键，$y_i$是值，核函数就是我们实现注意力机制的关键所在，在核函数作用下当查询和键越匹配的时候，给到对应的值也就越大；

根据Nadaraya-Watson核回归，我们不难实现一个简易的参数化、可学习的注意力机制网络

在操控注意力机制的 感知输入 与 键 的差值处引入可学习参数$\omega$:
$\hat{y} = \sum^n_{i=1}\frac{K((x-x_i)\omega)}{\sum^n_{j=1}K(x-x_j)}y_i$

模型定义如下

class NWKernelRegression(nn.Module):
    def __init__(self, **kwargs):
        super().__init__(**kwargs)
        self.w = nn.Parameter(torch.rand((1,), requires_grad=True))

    def forward(self, queries, keys, values):
        # queries和attention_weights的形状为(查询个数，“键－值”对个数)
        queries = queries.repeat_interleave(keys.shape[1]).reshape((-1, keys.shape[1]))
        self.attention_weights = nn.functional.softmax(
            -((queries - keys) * self.w)**2 / 2, dim=1)
        # values的形状为(查询个数，“键－值”对个数)
        return torch.bmm(self.attention_weights.unsqueeze(1),
                         values.unsqueeze(-1)).reshape(-1)
  1. 模型中的可训练参数为w，它是一个(1,)的张量，和我们在第二节讨论的理论类似，用于控制注意力的分布
  2. 这里使用的核函数是softmax函数，其与高斯核函数类似，当查询 接近 键 的时候 得到的注意力权重”attention_weights”会更大，最终在进行加权平均（由return后的批量矩阵乘法实现），键所对应的值会得到更大的权重
  3. 这与一般的网络结构不同 一般的网络中 我们通常是有一个input然后得到一个output 这里却是需要输入Q-K-V的三元组，然后得到一个output

注意，Q-K-V三元组的输入也是一个带注意力机制网络的特点，在后续的transformer中我们也会看到这点

至此我们可以从数学意义上给出前面带注意力机制的网络结构

网络的输入为Q-K-V三元组，其中

$Q (\in R^{50 \times 49}) = $
$
\begin{pmatrix}
x_0&\cdots&x_0\
\vdots&\ddots&\vdots\
x_{49}&\cdots&x_{49}
\end{pmatrix}
$

$K (\in R^{50 \times 49}) = $
$
\begin{pmatrix}
x_1&\cdots&x_{49}\
\vdots&\ddots&\vdots\
x_{0}&\cdots&x_{48}
\end{pmatrix}
$

$V (\in R^{50 \times 49}) = $
$
\begin{pmatrix}
y_1&\cdots&y_{49}\
\vdots&\ddots&\vdots\
y_{0}&\cdots&y_{48}
\end{pmatrix}
$

在forward前向传播中执行如下操作

def forward(self, queries, keys, values):
    # queries和attention_weights的形状为(查询个数，“键－值”对个数)
    queries = queries.repeat_interleave(keys.shape[1]).reshape((-1, keys.shape[1]))
    self.attention_weights = nn.functional.softmax(
        -((queries - keys) * self.w)**2 / 2, dim=1)
    # values的形状为(查询个数，“键－值”对个数)
    return torch.bmm(self.attention_weights.unsqueeze(1),
                     values.unsqueeze(-1)).reshape(-1)

网络的操作

  1. 通过softmax层构造注意力矩阵
  • 这里的softmax在dim=1上执行，即对每一行进行softmax，得到49个元素（类比之前构建softmax网络的时候，在进行softmax之前我们会有一个flatten的操作）
  • 此时得到的$50\times 49$的注意力矩阵$W$的意义可以这样理解：第i行代表根据$x_i$对$x_0,…,x_{i-1},x_{i+1},…,x_{49}$的距离分配对应的权重$w_0,…,w_{i-1},w_{i+1},…,w_{49}$
  1. 向注意力矩阵的维度索引1处插入一个新的维度，此时矩阵作为张量的形状发生了变化：(50,49)->(50,1,49)
  2. 向value矩阵的最后一维插入一个新的维度，张量形状变化：(50,49)->(50,49,1)
  • 到这个位置我们会发现，我们由Q-K构造了注意力矩阵，并将注意力矩阵与value矩阵转换为了前面批量矩阵乘法所需的张量的size
  1. 执行批量矩阵乘法(50,1,49)*(50,49,1)->(50,1,1)
  • 乘法的意义可以从单个sample的角度出发理解（因为我们知道前面一系列操作是为了让网络一次由一个batch进行训练，就像我们在线性回归、softmax回归中做的那样）
  • 仅看单个sample，例如在batch中sample索引为i时，对应元素$x_i$：1）它的注意力权重对应了$w_0,…,w_{i-1},w_{i+1},…,w_{49}$；2）权重用于对value的加权求和，即$y_0,…,y_{i-1},y_{i+1},…,y_{49}$；3）最终得到了一个(1,1)的张量，其中唯一的元素就是根据$x_i$对除$y_i$以外标签的加权求和

总结：

  1. 网络的作用基本与第二节中 带参数的Nadaraya-Watson核回归 类似，但是有两点不同：1）网络的训练是批量进行的，为了实现这个目的需要对数据做一些处理，从而得到Q-K-V三元组的网络输入；2）对于样本$i$的注意力权重分配到了除$i$以外所有的样本上，这样做的好处避免过拟合；
  2. 带注意力机制的网络使用Q-K-V输入是一种常态化的规范，模型使用Q-K构造W（注意力权重），以此来加权V得到该层网络的输出；
  3. 模型学习的参数存在于由Q-K构造W的过程中

关于总结中的1.2）参考下面内容

在 Nadaraya–Watson 核回归中，如果允许查询点使用自身作为键（即匹配自身），那么当带宽趋近于 0 时，每个查询点会完全依赖自身的标签，从而退化为一个记忆训练集的模型，无法泛化到新数据。即便带宽固定，自匹配也会导致模型倾向于过拟合，因为自身标签与其他样本的加权平均相比总是具有最小的距离（距离为零），从而获得最大的权重。

为了避免这种平凡解，并让模型真正学习到如何利用其他样本的信息进行插值，我们刻意排除自身。这样做相当于强制每个查询点只能从其他样本中“借鉴”知识，从而迫使模型学习到更平滑的、基于邻近性的估计。这种设计在统计学中对应“留一法”交叉验证的思想，也被广泛应用于非参数回归和注意力机制的变体中。

其中的带宽可以作为我们理解该回归的另一个视角：

  1. 带宽一词出自高斯核函数中的$\sigma$
  • $exp(\frac{-(q-k)^2}{2\sigma^2})$
  • 显然，前面模型学习的$\omega$从也可以理解为带宽
  • 带宽会控制核函数的尖锐、平缓程度，即注意力权重的分布情况
  • 考虑$\sigma$作为带宽，1）带宽很大：核函数很平缓，几乎所有键的权重都相近，预测值趋近于所有值的平均；2）带宽很小：核函数很尖锐，只有非常接近查询点的键才有显著权重，预测值更依赖于邻近点；
  1. 另一个视角就是，我们模型学习的本质是在学习哪种注意力权重的分布更加适合当前的数据集（更有利于一个输入样本（查询）从其他样本（键）中获取到有价值的信息，从而来预测对应的输出（值））
  • 从这种视角我们也能够看到这种方法的局限所在：模型学习时对注意力权重的分布控制是十分有限的，从高斯核函数的视角下来说我们仅学习一个带宽，即控制分布的尖锐程度；而分布更多是由我们选择的核函数本身——高斯分布，来决定的；更好的方法或许是选择更适合的分布，并且学习控制该分布的更多的参数，来得到更适合当前数据集的注意力分布

注意力评分函数

评分函数：上一节使用了高斯核来对查询和键之间的关系建模。其中高斯核函数的指数部分（计算查询和键的距离）可以视为注意力评分函数（attention scoring function）

函数的输出结果输入到softmax函数（当然也可以输入到高斯函数中）中进行运算。通过上述步骤，将得到与键对应的值的概率分布（即注意力权重）。

最后，注意力汇聚的输出就是基于这些注意力权重的值的加权和。

上述过程可以用下图表示

用一般的数学语言可以这样描述

  1. 有查询、键、值：$q\in R^q,k\in R^k,v\in R^v$
  2. 注意力汇聚后的结果为：$f(q,(k_1,v_1),…,(k_i,v_i),…,(k_m,v_m))=\sum_{i=1}^m \alpha(q,k_i)v_i \in R^v$
  3. 一种具体情况可以是$\alpha(q,k_i)=softmax(a(q,k_i))$
  4. 其中注意力评分函数为$a$，查询$q$和每个键$k_i$通过它将两个向量映射为标量的注意力权重（再经过softmax将权重归一化），最终用于对值进行汇聚

一些常用的注意力函数

  1. 加性注意力函数$a(q,k) = w^T_v \tanh(W_q q+W_k k) \in R$
  • 可学习的参数是$W_q\in R^{h\times q}, W_k\in R^{h\times k},w_v \in R^h$
  • 这实际上就是 将查询和键连结起来后输入到一个多层感知机（MLP）中，感知机包含一个隐藏层，其隐藏单元数是一个超参数$h$，使用了\tanh 作为激活函数，隐藏了偏置项
  • 这种注意力函数通常用于查询和键是不同长度的矢量时
  1. 缩放点积注意力函数$a(q,k) = q^T k/ \sqrt{d}$
  • 常用于查询和键是相同长度的矢量
  • 并且有假设：查询向量、键向量的点积的均值为$0$，方差为$d$

注意

  1. 上面都是注意力评分函数，最终会将查询和键映射为一个标量
  2. 实际情况通常会将这些标量再经过softmax处理归一化后再作为权重，并且实际情况下不会逐个样本的处理，而是通过批量矩阵乘法处理
  • 以缩放点积注意力为例子:注意力聚合后的结果为$softmax(\frac{QK^T}{\sqrt{d}}V)\in R^{n\times v},Q\in R^{n\times d},K\in R^{m\times d},V\in R^{m\times v}$（这里有n个查询、m个键、对应m个值，并且查询、键、值都被一般化到了某长度的向量）

    """通过在最后一个轴上掩蔽元素来执行softmax操作"""

        # 最后一轴上被掩蔽的元素使用一个非常大的负值替换，从而其softmax输出为0
        X = d2l.sequence_mask(X.reshape(-1, shape[-1]), valid_lens,
                              value=-1e6)
        return nn.functional.softmax(X.reshape(shape), dim=-1)

具体的功能是针对

  1. (batch_size, seq_len, d)的数据，以 1个batch 或 1个序列 为单位 指定特征的有效长度
  • 这里(batch_size, seq_len, d)的表述是数据的一般形式：1）batch_size不需多言，就是一个批次中sample的数量；2）seq_len即序列长度；3）d是特征维度；
  • 上面的“序列长度”与“特征维度”是抽象一点的表示，在具体的情况下有具体的含义，例如：1）自然语言处理，序列可以由token组成，一个token就是一个词，它会被映射为一个d=512的特征向量，然后一个序列中包含多个token，从而作为一个sample；2）如果处理图像的话，类似有将一个图像切成多个patch，一个patch就是一个序列，它会被平展为一个特征向量；3）还有类似对应一个序列的就是一个“时间步”，步内的信息会被作为一个特征向量处理
  1. 当valid_len是1D张量时，有效长度以batch为单位指定，该batch中的所有seq的所有特征向量，采取相同的有效长度
  2. 当valid_len是2D张量时，有效长度以seq为单位指定，一个seq中的不同特征相同采取不同的有效长度

  • 实现方式就是将原始的(batch_size,seq_len,d)的数据在最后一维（特征向量的维度），超出有效长度的部分设置为一个非常大的负数，这样softmax之后就几乎是0

多头注意力机制

前面我们在讨论注意力汇聚的时候，只涉及一次注意力的汇聚，这时可以理解为单头。

但是在实践中，当给定相同的查询、键和值的集合时，我们希望模型可以基于相同的注意力机制学习到不同的行为，然后将不同的行为作为知识组合起来，捕获序列内各种范围的依赖关系（例如，短距离依赖和长距离依赖关系）。

这时我们希望使用多头注意力机制；

其实现的思想是：

  • 我们可以用独立学习得到的$h$组不同的线性投影（linear projections）来变换查询、键和值。然后，这$h$组变换后的查询、键和值将并行地送到注意力汇聚中。
  • 最后，将这$h$个注意力汇聚的输出拼接在一起，并且通过另一个可以学习的线性投影进行变换，以产生最终输出。

这种结构如下图所示：

数学模型

  1. 给定查询、键、值$q\in R^{d_q},k\in R^{d_k},v \in R^{d_v}$
  2. 每个注意力头$h_i = f(W^{(q)}_i q,W^{(k)}_i k, W^{(v)}_i v) \in R^{(p_v)}$
  • 其中$f$就是代表注意力汇聚的函数，例如使用加性注意力、缩放点积注意力等
  • 此处可学习的参数包括$W^{(q)}_i ,W^{(k)}_i , W^{(v)}_i$，对应注意力头$h_i$之前，面向$q,k,v$的全连接层（线性变换）
  1. 多头注意力输出$h = W_o (h_1,\dots,h_h)^T \in R^{p_o}$
  • $W_o$是可学习参数，对应面向多头注意力输出的全连接层（线性变换）

这种设置下，每个头都可能关注输入不同的部分，可以表示比简单加权平均更加复杂的函数

自注意力和位置编码

自注意力

  • 有了注意力机制之后，我们将词元序列输入注意力池化中，以便同一组词元同时充当查询、键和值。
  • 具体来说，每个查询都会关注所有的键－值对并生成一个注意力输出。
  • 由于查询、键和值来自同一组输入，因此被称为 自注意力（self-attention）

位置编码

引：在处理词元序列时，循环神经网络是逐个的重复地处理词元的， 而自注意力则因为并行计算而放弃了顺序操作。如果我们想要使用序列的顺序信息的话，就可以使用位置编码

具体的方法：通过在输入表示中添加 位置编码（positional encoding）来注入绝对的或相对的位置信息。

固定位置编码（绝对位置信息）

  1. 输入$X\in R^{n\times d}$包含一个序列中$n$个词元的$d$维嵌入表示
  2. 位置编码使用相同形状的位置嵌入矩阵$P\in R^{n\times d}$
  3. 经过位置编码层后输出$X+P$，然后在作为原来$X$所担任的“输入”的角色，进行后续操作
  4. $X+P$相对与$X$的改变是，矩阵第i行、第2j列、第2j+1列上的元素是：$p_{i,2j} = \sin(\frac{i}{10000^{2j/d}}), p_{i,2j+1} = \cos(\frac{i}{10000^{2j/d}})$
  • 这样设置位置编码的原因是，随着列号(j)的增大，编码函数值随着行号(i)变化频率会越来越小
  • 类似于我们将信息编码为二进制01串后，高位的01变化相比于低位的01变化频率显然会慢很多（因此这样的绝对位置编码可以在这样的编码信息中反映出这种代表序列信息的“模式”，从而被网络学习到）
