hot1-两数之和

思路：

  • 使用哈希表存储value-index的索引
  • 两个需要关注的点：1）不能使用相同元素；2）哈希表value,index只能存储一个 元素值,元素索引 的pair
  • 如果提前for预处理哈希表可能引发的问题：在存在答案 or 不存在答案的情况下 都可能以重复的相同元素的索引作为答案返回（显然错误）
  • 解决方法：一次for循环，每检查一个x，检查target-x是否在哈希表中，在则找到答案；否则将 x,index 存入哈希表
  1. 不用担心返回两个相同元素的索引，检查到x时可以确定 x,index 不在哈希表中
  2. 不用担心有相同值的元素在哈希表中的存储被覆盖，因为只用返回一组有效答案

知识点：

  • 使用map<int,vector> m容器，直接访问m[0]，会调用vector默认构造函数，初始化一个空容器
  • 实际上map不能称为哈希表，它是一个字典，底层是红黑树实现的，查找效率是O(logn)，而非1
  • unordered_map才是哈希表

反思：

  • 想到了使用哈希表，但是简单粗暴的在开始解决问题之前，通过for做了一遍完成的预处理
  • 问题是对于重复值作为key，要索引到它们在nums唯一的index的处理，解决方法是vector
  • 通过vector让哈希表实现一个key对应多个index，但这样相比标答的速度会慢一些
class Solution {
public:

    map<int,vector<int>> book;//存放nums的 值->索引 由于值可能有相同的，索引唯一，所以map的value使用vector
    vector<int> ans;
    vector<int> twoSum(vector<int>& nums, int target) {
        for(int i = 0;i < nums.size();i++)
            book[nums[i]].push_back(i);
        for(int i = 0;i < nums.size();i++){
            int to_find = target - nums[i];//需要找的值
            int to_index = -1;
            if(!book[to_find].empty()){//如果该值对应的索引容器不为空
                for(int j = 0;j < book[to_find].size();j++)
                {
                    if(book[to_find][j] != i){
                        to_index = book[to_find][j];
                        break;
                    }
                }
            }
            if(to_index != -1){
                ans.push_back(i);
                ans.push_back(to_index);
                break;
            }
        }
        return ans;
    }
};

hot2-字母异位词分组

思路：

  • 和hot-1类似，使用哈希表，一次for就可以解决
  • 因为要把字母异位分词放在同一组中，哈希的构造肯定是 key = 字变异位分词的共同点, value = 分组的索引
  • 后者很简单，关键是考虑共同点，有三种方法
  1. 给字符串排序，这是很容易想到的，因为字母异位词排序后肯定是相同字符串
  2. 使用对字母计数的列表作为key，由于没有排序，时间会快一些，但是unorderd_map没有默认对“列表”的哈希实现，需要自定义哈希函数（不推荐）或者
  3. 作为2、的替代，仍然是计数，但是将数字编码程字符串作为key

知识点：

  • stl标准库的数组array
  1. 初始化: array<type_name,size> a;
  2. 常用的操作：增、删（不能，因为大小固定），改（使用[]，进行访问即可）
  • 使用map只要key支持 operator < 即可；
  • 使用unordered_map需要key支持哈希；

思路1:

class Solution {
public:
    vector<vector<string>> ans;
    vector<string> bf;
    map<string,int> m; //记录某str应该放在ans的index
    vector<vector<string>> groupAnagrams(vector<string>& strs) {
        bf = strs;
        for(int i = 0;i < strs.size();i++){
            sort(strs[i].begin(),strs[i].end());

            if(m.find(strs[i]) == m.end()){
                vector<string> t;
                t.push_back(bf[i]);
                ans.push_back(t);
                m[strs[i]] = ans.size()-1;
            }else{
                int index = m[strs[i]];
                ans[index].push_back(bf[i]);
            }  
        }
        return ans;

    }
};

hot3-最长连续序列

思路1：

  • 先使用sort，原始数组进行排序
  • 维护一个vector存放当前的连续序列，遍历原始数组更新它
  • 在每次连续序列断开的时候，对最大长度ans进行更新，遍历到i时三种情况
  1. i对应的数等于i-1对应的数+1，将该i对应的数插入vector；
  2. i对应的数等于i-1对应的数，不做处理
  3. 其他情况，判断发生“断开”
  • 注意，由于原始数组最后一个位置的数不一定导致断开，所以要添加一个特判，即在末尾检查ans是否可以更新

思路2: (思路1时间复杂度是O(nlogn)，思路2可以实现o(n))

  • 使用哈希表将原始数组的所有“(值，索引)”作为键值对存储起来
  • for遍历原始数组，两种情况：
  1. 发现当前当前数组值-1 的值存在于哈希表中，则不进行处理；
  2. 发现当前数组值-1 的值不存在于哈希表中，使用哈希表不断遍历当前 数组值+1 的值对应的键值对，直到没有对应的键值对
    （每完成一次哈希表上的遍历则更新一次答案，由于情况1的跳过处理，最终对于长度为n的数组实际上被遍历了2n次，时间复杂度是O(n)）

思路1:

class Solution {
public:
    
    int longestConsecutive(vector<int>& nums) {
        sort(nums.begin(),nums.end());
        int ans = 0;
        vector<int> t2;
        
        for(int i = 0;i < nums.size();i++){

            if(t2.size() == 0){
                t2.push_back(nums[i]);
                if(ans == 0)ans = 1;
                continue;
            }

            if(i == nums.size()-1){
                //末尾没有中断的情况特殊处理
                //因为如果末尾没有中断，无法检查ans是否需要更新
                if(nums[i]-1 == t2[t2.size()-1]){
                    if(t2.size()+1 > ans)ans = t2.size()+1;
                    break;
                }
                if(t2.size() > ans)ans = t2.size();
            }

            if(nums[i]-1 == t2[t2.size()-1]){
                t2.push_back(nums[i]);
                continue;
            }else if(nums[i] == t2[t2.size()-1]){
                continue;
            }else{
                if(t2.size()>ans)ans = t2.size();
                i--;
                t2.clear();
            }
        }
        return ans;
    }
};

class Solution {
public:
    
    int longestConsecutive(vector<int>& nums) {
        unordered_map<int,int> m;
        int ans = 0;
        for(int i = 0;i < nums.size();i++)
            m[nums[i]] = i;//这样处理也等价于对nums进行了去重，对于相同值保留的是出现在后面的
        for(auto ite = m.begin();ite != m.end();ite++){
            if(m.find(ite->first - 1)!= m.end()){
                continue;
            }else{
                int count = 0;
                int index = ite->second;
                while(1){
                    count++;
                    if(m.find(nums[index]+1) == m.end())break;
                    else index = m[nums[index] + 1];
                }
                if(count > ans)ans=count;
            }
        }
        return ans;
    }
};

hot4-移动零

要求不能复制数组，在原地进行操作

思路：

  • 使用双指针，左指针指向处理好的序列的尾部，右指针指向未处理的序列的头部
  • 右指针向右遍历，每遍历一个数字则代表该数字处理过了，有两种情况
  1. 如果数字为0则右指针向右继续遍历+1，左指针不动，可以理解为此时数字0就在“处理过的序列”的末尾
  2. 如果数字不为0则右指针指向的数字与左指针指向的数字交换，然后再右指针+1、左指针+1，因为此时左指针指向的是“处理过的序列”的末尾，可以理解为该末尾就是0（不是0的情况一定有左右指针相同，相当于没有交换数字的位置），现在处理非0数字的时候，就是将它放在0的前面；
    （整个过程相当于遍历一遍数字找到非0的数然后插入处理过的序列，只是需要一个左指针来保存下一个插入的位置应该在哪）
class Solution {
public:
    void moveZeroes(vector<int>& nums) {
        for(int r,l = 0;r < nums.size();r++){
            if(nums[r] == 0){
                continue;
            }else{
                int t = nums[r];
                nums[r] = nums[l];
                nums[l] = t;
                l++;
            }
        }
    }
};

最初的思路：

  • 使用双指针，外层i遍历原始数组，如果遇到0的话
  • 内层j向前遍历，将所有后面的数字向前移动一位，然后将最后一个数字置为0
  • 有两个需要注意的点：
  1. 如果处理完遇到0的情况，指针i应该减1，否则下一次for循环i+1相当于直接跳过了原本的下一个数（因为所有数字向前移动了一位）
  2. 在注意点的基础上，我们需要附加额外的条件判断外层循环何时结束（因为遍历到最后，所有0都移动到了末尾，按照刚才到逻辑i每次都会减1，会一直卡在循环里面无法退出）；附加的额外条件就是i == size() - count;//count是处理过的0的数量；

但是这样的时间复杂度实际上是$O(n^2)$，而标准思路的时间复杂度只有$O(n)$

class Solution {
public:
    void moveZeroes(vector<int>& nums) {
        int count = 0;//注意点2
        for(int i = 0;i < nums.size();i++)
        {
            if(i == nums.size()-count)break;
            cout<<i<<endl;
            if(nums[i] == 0){
                for(int j = i;j < nums.size()-1;j++){
                    nums[j] = nums[j+1];
                }
                nums[nums.size()-1] = 0;
                i--;//注意点1
                count++;//注意点2
            }
        }
    }
};

hot5-盛最多水的容器

思路：

  • 使用双指针r、l，指向当前数组的首、末位置
  • 双指针对应的闭区间代表了当前可作为容器边界的可选范围，而双指针自己本身当前指向的就是“当前选中的容器边界”
  • 每次都移动height较小的对应指针，缩小“可选范围”（因为移动hegiht较大的指针，得到的新容器无论如何容积也不可能大于当前容器）
  • 每次移动的指针，移动到该指针的height大于原始值时尝试更新ans（或者也可以直接尝试更新，不会多多少时间）

这种标准做法的时间复杂度是$O(n)$

另一种暴力做法的视角，实际上就是枚举数组元素的所有组合，每个组合一个衡量值，选择值最大的即可。标准做法相对于这种做法的好处就在于利用了作为容器的边界的位置，一定是一左一右的，从而省去了暴力枚举的很多“重复情况”；

class Solution {
public:
    int maxArea(vector<int>& height) {
        int l = 0,r = height.size()-1;
        int ans = (r-l)*min(height[r],height[l]);
        while(l < r){
            if(height[l] < height[r]){
                l++;
                ans = max(ans,(r-l)*min(height[r],height[l]));
            }else{
                r--;
                ans = max(ans,(r-l)*min(height[l],height[r]));
            }
        }
        return ans;
    }
};

hot6-三数之和

思路：（比较困难的地方是如何实现不重复的需求，尤其是在不使用额外的stl容器，如set、unordered_map等）

  1. 去重的方法，要求找到的三个数字$a,b,c$满足$a\le b \le c$，因为重复的本质原因是找到了$a,b,c$又找到了$b,a,c$或$c,a,b$等等
  • 我们不能要求$a\l b\l c$作为条件，因为题目允许不同位置的相等元素被采纳，最好的例子就是$0,0,0$（注意三个0来自不同位置）
  • 但是这意味着$a\le b\le c$的条件是不能严格去重的，例如$b_1,b_2,c$和$b_2,b_1,c$（这里$b_1 = b_2$），就是重复的
  • 但是上面的情况可以在实际操作时 结合排序 加入一些trick来处理
  1. 对原始数组排序（在所选元素有大小要求的条件下，排序之后往往能让我们处理更简单）
  2. 维护对应前面三个数字的三个指针a、b、c
  • a从0开始向后遍历，作为一重循环
  • b从a+1开始向后遍历，作为二重循环
  • c从末尾向前遍历，作为三重循环，但是c并非每次b重置之后就重置的，相反其一直被维护着，并且会作为二重循环的提前退出条件：c==b，则二重循环退出直接
  • 排序之后的这种三重循环枚举可以保证得到的满足条件的组合，只有不严格单调递增的结果
  • trick：第1、2重循环之后检查当前处理的数字是否与上一个数字相同，如果是的话则跳过。
    （第1重循环跳过一次的原因是，对于指定的相同的target的两数之和，一次遍历就可以找到所有满足条件的组合）
    （第2重循环跳过一次的原因是，在两数之和的问题下，一个数字对应当前target一定只有一个满足条件的数，如果存在的话在上一次循环中一定已经找到了）
    （注意2、3重循环本质上是并列的，寻找满足一个target的两数之和，尤其是在有序的情况下，实际上就是一次O(N)的遍历）
    （对指针的操作就类似于，两数之和大于target就移动c，让和变小，和大于target就移动b，让和增大，如果刚好等于target就保存结果，然后看是否能继续尝试（移动b、c都行，在实现中可以规定移动b））

一个结论：在有序数组中枚举两个满足和等于某数的元素值（不需要重复值），可以使用双指针左右两边并列枚举，实现O(N)时间复杂度的枚举

知识点：

  1. C++中set可以对vector去重
  • set对去重条件是要求元素能够实现“严格弱序”，即元素支持operater <
  • vector容器实现了，容器之间对于operater <按照字典序进行比较
class Solution {
public:
    vector<vector<int>> threeSum(vector<int>& nums) {
        vector<vector<int>> ans;
        sort(nums.begin(),nums.end());
        for(int a = 0;a < nums.size();a++){
            //跳过一次第一重循环
            if((a != 0) && (nums[a] == nums[a-1]))continue;
            
            int target = 0 - nums[a],c = nums.size()-1;
            for(int b = a+1;b < nums.size();b++){//二重循环相当于在移动左指针
                //跳过一次第二重循环
                if((b != a+1)&&(nums[b] == nums[b-1]))continue;

                //两数之和大于target，移动右指针
                while(nums[b]+nums[c] > target && c>b)c--;
                
                //注意这里不能紧接着上一个if使用else
                //否则就是移动了右指针但是没有检查是否满足条件又移动左指针，可能就会漏情况
                if(nums[b]+nums[c] == target && c>b){
                    vector<int> t = {nums[a],nums[b],nums[c]};
                    ans.push_back(t);
                }

                if(c<=b)break;//两数之和并列循环的退出条件
            }
        }
        return ans;
    }
};

hot7-接雨水

最初的思路：

  1. 按照某个设定的高度遍历当前的柱子，找到两个东西：
  • 当前高度的边界
  • 当前高度除了边界的柱子还有哪些位置没有柱子
  1. 当前高度边界范围内，没有柱子的位置就是可以有雨水的位置
  2. 外层遍历所有可能的高度（结束条件是某个高度只有一个边界）

时间复杂度应该是O(nm)，其中n是柱子数量，m是高度数量

class Solution {
public:
    int trap(vector<int>& height) {
        int max_h = 0,ans = 0;
        for(int i = 0;i < height.size();i++)
            if(height[i]>max_h)max_h = height[i];
        if(max_h == 0)return 0;

        for(int h = 1;h <= max_h;h++){
            int l = -1,r = -1,cnt = 0;
            for(int i = 0;i < height.size();i++){
                if(height[i] >= h){
                    if(l == -1)l = i;//左侧边界是遇到的第一个非0高度
                    r = i;//右侧边界不断增大
                    cnt++;//当前层有效的柱子数量
                }
            }

            if(l == r)break;//搜索结束
            else{
                ans += (r-l+1-cnt);
            }
        }
        return ans;
    }
};
